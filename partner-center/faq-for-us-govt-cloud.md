# <a name="frequently-asked-questions-for-partner-center-for-microsoft-cloud-for-us-government"></a>Często zadawane pytania dotyczące Partner Center dla Microsoft Cloud for US Government

**Dotyczy**

- Centrum partnerskie Microsoft Cloud for US Government

**Odpowiednie role**

## <a name="general-questions"></a>Pytania ogólne

Użyj tych często zadawanych pytań, aby dowiedzieć się więcej na temat Dostawca rozwiązań w chmurze (CSP) dla Microsoft Cloud for US Government. Aby uzyskać więcej informacji, możesz również skontaktować się z menedżerem konta.

### <a name="what-is-microsoft-cloud-for-us-government"></a>Co to jest Microsoft Cloud for US Government?

Zaprojektowana dla instytucji rządowych USA od podstaw, usługa Microsoft Cloud for US Government umożliwia klientom z sektora publicznego w Stany Zjednoczone — od dużych agencji federalnych po instytucje rządowe małych miast — wybór z wielu usług przetwarzania w chmurze. Aby spełnić określone potrzeby klientów, firma Microsoft pozyła znaczące inwestycje w centra danych i jest przeznaczona do zapewnienia zgodności z amerykańskimi zasadami federalnymi i stanowymi, uprawnieniami i wymaganiami.

### <a name="who-is-eligible-for-the-csp-program-for-microsoft-cloud-for-us-government"></a>Kto kwalifikuje się do programu CSP na Microsoft Cloud for US Government?

Partnerzy w organizacji Stany Zjednoczone, którzy służą federalnym, stanowym i lokalnym jednostkom rządowym w ramach swoich instytucji rządowych, są uprawnieni do korzystania z programu CSP dla Microsoft Cloud for US Government. Firma Microsoft dysponuje rygorystycznym programem weryfikacji w celu określenia uprawnień, zanim partnerzy będą Microsoft Cloud for US Government. Aby się zakwalifikować, musisz udowodnić, że pracujesz z klientami rządowymi. Proces rejestracji w programie CSP w programie Microsoft Could for US Government został rozszerzony w celu przechwycenia większej liczby danych w celu przeprowadzenia walidacji. Typy dowodów mogą obejmować między innymi numery umów dla instytucji rządowych i list sponsorowania od klienta rządowego.

### <a name="is-customer-pricing-the-same-as-for-csp-commercial"></a>Czy ceny klientów są takie same jak w przypadku usług CSP Commercial?

Nie. Ceny będą odzwierciedlać dodatkowe koszty w celu zapewnienia unikatowej wartości Microsoft Cloud for US Government.

### <a name="is-the-partner-discounting-model-the-same"></a>Czy model rabatów partnera jest taki sam?

Tak. Model rabatów partnera dla programu CSP Microsoft Cloud for US Government jest taki sam jak model rabatów partnera dla programu CSP Commercial.

### <a name="csp-commercial-is-an-existing-partner-program-how-is-the-csp-program-for-microsoft-government-cloud-different"></a>Program CSP Commercial to istniejący program partnerski. Czym różni się program CSP dla chmury microsoft government?

Program CSP dla Microsoft Cloud for US Government jest inny niż CSP Commercial w następujących obszarach:

- Uprawnienia: Program CSP dla programu Microsoft Cloud for US Government jest dostępny tylko dla partnerów w Stany Zjednoczone, którzy obsługują federalne, stanowe i lokalne jednostki rządowe USA w ich ramach rządowych. Firma Microsoft ma program weryfikacji w celu określenia uprawnień, zanim partnerzy będą mieć dostęp do chmury dla instytucji rządowych USA.

- Dołączanie: Partnerzy muszą do dołączać do programu CSP Microsoft Cloud for US Government oddzielnie od programu CSP Commercial.

- Dostępność produktu: usługi Azure Government i oparte na licencjach, takie jak Office 365, Enterprise Mobility + Security i Dynamics 365, są dostępne w programie CSP dla Microsoft Cloud for US Government.

- Umowa odsprzedawcy chmury firmy Microsoft (MCRA) została zaktualizowana w celu odzwierciedlenia unikatowych warunków dla instytucji rządowych.

- Partner Center funkcjonalności: niektóre funkcje programu Partner Center nie będą włączone dla programu CSP dla Microsoft Cloud for US Government spełniania wymagań prawnych i zgodności. Partnerzy mogą zamiast Azure Active Directory (Azure AD) lub [interfejs Graph API](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS) do wykonywania zadań związanych z zarządzaniem użytkownikami i grupą. Funkcja żądania relacji do obsługi scenariuszy wielokanałowych i między partnerami będzie dostępna po uruchomieniu, gdy usługi oparte na licencjach będą dostępne w programie CSP dla Microsoft Cloud for US Government.

### <a name="i-am-not-a-csp-partner-yet-where-can-i-find-more-information-on-how-to-become-a-csp-partner"></a>Nie jestem jeszcze partnerem programu CSP. Gdzie można znaleźć więcej informacji o tym, jak zostać partnerem CSP?

Aby dowiedzieć się więcej na temat modeli rejestracji dostępnych dla partnerów, [zobacz Dostawca rozwiązań w chmurze na Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider).

### <a name="how-does-licensing-work"></a>Jak działa licencjonowanie?

*Umowa odsprzedawcy chmury firmy Microsoft (MCRA)*

Umowa MCRA zezwala partnerom na odsprzedawcę przy użyciu programu CSP. Wersja tej umowy w chmurze dla instytucji rządowych USA jest akceptowana podczas dołączania do usługi Microsoft Cloud for US Government. Ta wersja zawiera dodatek do chmury dla instytucji rządowych USA i załącznik A po standardowej umowie odsprzedawcy chmury.

McRA for Microsoft Cloud for US Government różni się od standardowej mcra na następujące dwa sposoby:

- Dodatek do chmury dla instytucji rządowych USA zawiera dodatkową odpowiedzialność partnera za transakcję usług w chmurze dla instytucji rządowych USA.

- Załącznik A zawiera Umowa z Klientem Microsoft (Umowa z Klientem Microsoft), które należy włączyć do każdej umowy dotyczącej usług w chmurze dla instytucji rządowych USA, która została zawarta między programem CSP i jego klientem.

*Umowa z Klientem Microsoft (Umowa z Klientem Microsoft) prezentowane jako Załącznik A w MCRA*

CSP są wymagane, aby uwzględnić załącznik A w umowie dla instytucji rządowych zawartej między CSP i klientami, którzy Microsoft Cloud for US Government. Umowa jest publikowana na stronie Partner Center i jest dostępna w języku angielskim.

Umowa z Klientem Microsoft dla chmury dla instytucji rządowych USA różni się od standardowej Umowa z Klientem Microsoft na następujące dwa sposoby:

- Musi on zostać uwzględniony w umowie CSP ze swoim klientem.

- Został on dostosowany do zgodności z przepisami federalnymi, stanowymi i lokalnymi.

*Microsoft Cloud Distributor Agreement (MCDA)*

Umowa Microsoft Cloud Distributor Agreement (MCDA) zezwala dystrybutorom na sprzedaż za pośrednictwem dostawcy CSP. Wersja US Government Cloud umowy MCDA ma dodatek do chmury dla instytucji rządowych USA i załącznik A po standardowej umowie odsprzedawcy chmury firmy Microsoft (MCRA).

Umowa MCDA dla Microsoft Cloud for US Government różni się od standardowej mcda na następujące dwa sposoby:

- Microsoft Cloud for US Government dodatek zawiera dodatkową odpowiedzialność partnera za transakcję usług w chmurze dla instytucji rządowych USA.

- Załącznik A zawiera Umowa z Klientem Microsoft (Umowa z Klientem Microsoft), które należy włączyć do każdej umowy dotyczącej usług w chmurze dla instytucji rządowych USA zawartej między programem CSP i jego klientami.

### <a name="what-are-the-qualifications-to-sell-microsoft-cloud-for-us-government"></a>Jakie są kwalifikacje do sprzedaży Microsoft Cloud for US Government?

Kryteria kwalifikacji do sprzedaży programu CSP dla firmy Microsoft Cloud for US Government są takie same jak kryteria kwalifikacji dla programu CSP Commercial, otwarte zarówno dla partnerów bezpośrednich, jak i odsprzedawców pośrednich. Nie trzeba być wywłaszczynym odsprzedawcą pośrednim programu CSP dla programu CSP Microsoft Cloud for US Government jeśli został zatwierdzony jako odsprzedawca pośredni dla programu CSP Commercial.

### <a name="i-am-an-existing-partner-what-do-i-need-to-do-to-sell-microsoft-cloud-for-us-government"></a>Jestem istniejącym partnerem. Co należy zrobić, aby sprzedawać Microsoft Cloud for US Government?

Partnerzy będą musieli dopiąć program CSP, aby Microsoft Cloud for US Government spełniać wymagania prawne i zgodności. (Dołączanie dla Microsoft Cloud for US Government jest oddzielone od dołączania dla komercyjnego CSP). Po dostawiu możesz tworzyć klientów i oferty transakcji dostępne w ramach Microsoft Cloud for US Government. Jeśli używasz interfejsów API, zobacz Developing for Partner Center for Microsoft National Cloud (Tworzenie aplikacji [dla chmury microsoft national cloud).](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS)

### <a name="is-there-any-difference-in-partner-incentives"></a>Czy istnieją jakieś różnice w zachętach dla partnerów?

Nie. Zachęty partnerów dla programu CSP dla Microsoft Cloud for US Government są takie same jak zachęty partnerów dla programu CSP Commercial. Aby dowiedzieć się więcej na temat zachęt dla partnerów, zaloguj się do [Microsoft Partner Network portal.](https://partner.microsoft.com/membership/partner-incentives)

### <a name="are-there-any-changes-to-billing"></a>Czy istnieją jakieś zmiany w rozliczeniach?

Nie. Kryteria rozliczeniowe i wymagania dotyczące programu CSP dla programu Microsoft Cloud for US Government są takie same jak kryteria rozliczeniowe i wymagania dotyczące programu CSP Commercial. Aby dowiedzieć się więcej na temat rozliczeń, zapoznaj się z zasobami dostępnymi [Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider/resources).

### <a name="for-data-hosted-in-commercial-data-centers-is-it-possible-to-change-to-microsoft-cloud-for-us-government"></a>Czy w przypadku danych hostowanych w komercyjnych centrach danych można zmienić Microsoft Cloud for US Government?

W celu ułatwienia i utrzymania integralności izolacji Microsoft Cloud for US Government nie ma mechanizmów przenoszenia istniejących usług klienta do Microsoft Cloud for US Government. Jeśli klient lub partner ma platformę Azure, będzie on odpowiedzialny za przeniesienie danych do nowego środowiska w Microsoft Cloud for US Government.

## <a name="questions-about-microsoft-cloud-for-us-government-for-azure"></a>Pytania dotyczące Microsoft Cloud for US Government platformy Azure

### <a name="what-is-microsoft-cloud-for-us-government-for-azure"></a>Co to jest Microsoft Cloud for US Government platformy Azure?

Azure Government cloud to platforma w chmurze zbudowana na podstawowych zasadach zabezpieczeń, prywatności i kontroli, zgodności i przejrzystości. Jednostki sektora publicznego otrzymują fizycznie odizolowane wystąpienie usługi Microsoft Azure, które wykorzystuje światowej klasy usługi zabezpieczeń i zgodności krytyczne dla instytucji rządowych USA dla wszystkich systemów i aplikacji zbudowanych na podstawie jej architektury. Usługi te obejmują certyfikaty zgodności FedRAMP i DoD, umowy na poziomie stanu CJIS, możliwość wydawania umów HIPAA Business Associate oraz pomoc techniczną dla IRS 1075. Obsługiwane przez zweryfikowane osoby z USA, Azure Government obsługuje wiele scenariuszy hybrydowych tworzenia i wdrażania rozwiązań lokalnie lub w chmurze. Jednostki sektora publicznego mogą również korzystać z błyskawicznej skalowalności i gwarantowanego czasu pracy usługi w chmurze w hiperskalowaniu.

### <a name="does-the-csp-program-for-microsoft-cloud-for-us-government-offer-all-azure-services"></a>Czy program CSP dla Microsoft Cloud for US Government oferuje wszystkie usługi platformy Azure?

Infrastruktura jako usługa (IaaS) oraz podstawowe oferty platform i danych są teraz dostępne. Harmonogram działania jest aktualizowany na bieżąco, a w ramach tej usługi dostarczamy dokumentację podobną do dokumentacji dotyczącej programu CSP Commercial Partner Center (wymagane jest uwierzytelnianie). Możesz przejrzeć usługi Azure Government obecnie dostępne tutaj, ale należy pamiętać, że niektóre usługi, takie jak klasyczne usługi obliczeniowe, sieciowe, magazynowe i w chmurze, nie są przeznaczone dla dostawców CSP i nie działają.

### <a name="what-do-i-need-to-do-to-develop-with-the-csp-program-for-microsoft-cloud-for-us-government"></a>Co należy zrobić, aby programować za pomocą programu CSP dla Microsoft Cloud for US Government?

Aby uzyskać więcej informacji na temat tworzenia aplikacji przy Partner Center for Microsoft Cloud for US Government, zobacz Developing with Partner Center for Microsoft Government Cloud (Tworzenie aplikacji przy użyciu Partner Center [dla chmury Microsoft Government Cloud).](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS)

### <a name="are-there-any-differences-in-apis"></a>Czy istnieją jakieś różnice w interfejsach API?

Interfejsy API i wymagania dotyczące integracji programu CSP dla programu Microsoft Cloud for US Government są identyczne z interfejsami API i wymaganiami integracji dla komercyjnego programu CSP. Aby uzyskać więcej informacji na temat punktów końcowych w Microsoft Cloud for US Government, zobacz Partner Center REST URL (Adresy [URL REST).](/partner-center/develop/partner-center-rest-urls)

>[!NOTE]
>Interfejsy API CREST nie będą dostępne dla programu CSP dla Microsoft Cloud for US Government. Te interfejsy API są przestarzałe w programie CSP i dlatego nie są dostępne dla Microsoft Cloud for US Government. Aby spełnić wymagania prawne i wymagania dotyczące zgodności, następujące interfejsy API nie będą dostępne w programie CSP dla Microsoft Cloud for US Government.

>- Aktualizowanie kont użytkowników dla klienta
>- Pobieranie listy wszystkich kont użytkowników dla klienta
>- Resetowanie hasła użytkownika dla klienta
>- Pobieranie ról użytkowników dla klienta
>- Ustawianie ról użytkownika dla klienta
>- Wyświetlanie użytkowników usuniętych dla klienta
>- Przywracanie usuniętego użytkownika dla klienta
>

>- Pobieranie listy dostępnych licencji

Partnerzy mogą używać programu Azure Portal, programu Azure AD PowerShell lub interfejs Graph API do zarządzania kontami użytkowników partnera. Aby uzyskać więcej informacji technicznych, zobacz Developing with Partner Center for Government Cloud (Tworzenie rozwiązań dla chmury dla instytucji [rządowych w chmurze](https://msdn.microsoft.com/partner-center/partner-center-for-microsoft-us-govt-cloud)dla instytucji rządowych).
